## Design Principles

### **Composition over Configuration**
We favor composable components over props-heavy APIs. Use compound components like `Card.Header` and `Card.Body` instead of passing everything as props.

### **Semantic HTML**
Components use appropriate HTML elements by default (`<button>`, `<nav>`, `<header>`) but allow customization via the `as` prop when needed.

### **BoxProps Pattern**
Most components accept spacing props (padding, margin, gap, width, height) through the BoxProps system, enabling flexible layouts without custom CSS.

### **TypeScript First**
All components are fully typed with strict TypeScript. Props extend native HTML element props, so you get full autocompletion and type safety.

## Contributing

When creating new components in Ratio UI:

1. **Follow the pattern**: Look at existing components for structure
2. **Extend native props**: Use `ComponentProps<'button'>` etc.
3. **Add BoxProps**: Include spacing props when appropriate
4. **Ensure accessibility**: Use semantic HTML and ARIA where needed
5. **Document with Storybook**: Create stories showing different variants
6. **Type everything**: Use strict TypeScript

See [UI Component Guidelines](https://github.com/losol/eventuras/blob/main/.github/instructions/ui-components.instructions.md) for detailed instructions.
